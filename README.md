# Artemis II Live Transcript Archive

This repository stores transcript and analysis files generated from the official NASA YouTube live streams that covered the Artemis II mission.

The goal of this project is to preserve, organize, and make understandable the spoken content transmitted during this historic mission, so that even people who were not watching live can later understand what happened, what was discussed, and what technical situations occurred throughout each transmission window.

## What this repository contains

The transcript files in this repository were generated by a **BOT** that captured live audio from the official NASA YouTube streams and then sent the captured speech segments to **OpenAI models** for transcription.

The source material came from the official NASA YouTube live broadcasts during all days of mission transmission, especially from streams such as:

- `NASA's Artemis II Live Views from Orion`
- `NASA's Artemis II Live Mission Coverage`

The BOT continuously listened to the live audio, identified speech segments, and generated transcript files in one or more languages.

## Folder structure

This repository is organized into two main folders, each one named after the title of the NASA YouTube stream:

- `NASA's Artemis II Live Views from Orion`
- `NASA's Artemis II Live Mission Coverage`

Inside each folder, files follow a standard naming convention.

## File naming standard

Examples:

- `01 - Day 1 - 02-04-2026 - 2246-Transcript-EN.txt`
- `02 - Day 1 - 02-04-2026 - 2252-Transcript-EN.txt`

### Naming logic

Each file name follows this structure:

`01 - Day 1 - 02-04-2026 - 2246-Transcript-EN.txt`

Where:

- `01` = sequential file number
- `Day 1` = mission elapsed day reference
- `02-04-2026` = date in `dd-mm-yyyy`
- `2246` = start time of the recording block in `HHMM`
- `Transcript` = file type
- `EN` = transcript language

Other language versions may also exist, including:

- `PT-BR`
- `ES`

Examples:

- `03 - Day 2 - 03-04-2026 - 0815-Transcript-PT-BR.txt`
- `04 - Day 2 - 03-04-2026 - 0930-Transcript-ES.txt`

## Timestamp format inside transcript files

Inside transcript files, each entry includes a timestamp in the following format:

`05/04/2026 - 00:18:35 (-3 TMZ) | 03:04:43:35 (Artemis Clock)`

### Meaning of each part

- `05/04/2026 - 00:18:35` = local Brazil date and time
- `(-3 TMZ)` = Brazil time zone reference used in this project
- `03:04:43:35 (Artemis Clock)` = mission elapsed time in the format `DD:HH:MM:SS`

### Artemis Clock

The Artemis Clock is based on **Mission Elapsed Time (MET)** and is represented as:

- `DD` = days since mission start
- `HH` = hours
- `MM` = minutes
- `SS` = seconds

This makes it easier to understand not only the local time in Brazil, but also where that spoken event happened within the mission timeline.

### Time zone note

The repository uses **TMZ -3 (Brazil)** as the local timestamp reference.

People in other countries can convert the local time to their own time zone if needed. The Artemis Clock is especially useful because it is independent of local geography and remains tied to the mission timeline itself.

## How the BOT captured the transcripts

The BOT was designed to:

1. connect to an official NASA YouTube live stream
2. capture the live audio
3. detect when someone was speaking
4. wait for a silence window to determine the end of a speech segment
5. send that audio segment to OpenAI models for transcription
6. optionally translate the transcript into other languages
7. generate technical and risk analysis files from the transcript history

## Important accuracy note

Because this is an automated process, it is **not guaranteed** that every spoken sentence was captured perfectly.

Possible limitations include:

- background audio or mission audio mixing
- overlapping speakers
- partial phrases
- occasional missed words
- segments cut at imperfect moments
- model interpretation errors

## About silence-based segmentation

The BOT used a silence-based rule to decide when one speech segment had ended.

In simple terms:

- while a person was speaking, the BOT kept recording
- when there was a silence long enough, the BOT considered that speech finished
- then it sent that captured block for transcription

However, if another person started speaking **before enough silence had occurred**, both speeches could end up in the **same transcript block**.

So, in practice, sometimes two close back-to-back statements may appear together in one transcript entry instead of being separated into two lines. This is a normal consequence of automated segmentation and should be considered when reviewing the text.

## Analysis files

Files ending with names such as:

- `Analysis-EN.txt`
- `Analysis-PT-BR.txt`
- `Analysis-ES.txt`

contain **technical and risk analysis** related to the events captured during that recording period.

These analysis files were generated by **OpenAI models** based on the transcript history collected by the BOT.

The purpose of these files is to help identify:

- probable mission phase
- operational context
- successful actions
- successful actions with challenges
- technical anomalies
- risks and concerns mentioned during the period
- likely next steps in mission operations

## Purpose of the archive

This project was created to preserve and study a historic human spaceflight event in a structured and searchable format.

By capturing the official live mission audio and transforming it into text, translations, and technical summaries, the repository helps make the Artemis II mission easier to revisit, analyze, and understand over time.

## Disclaimer

This repository is an independent archival and analysis effort based on publicly available official NASA live streams.

It is not an official NASA transcript repository.

The generated transcripts and analyses are automated outputs and may contain omissions, merged statements, transcription inaccuracies, or interpretation differences.

Users should treat the files as a high-value reference archive, but not as a perfect verbatim or official mission record.
